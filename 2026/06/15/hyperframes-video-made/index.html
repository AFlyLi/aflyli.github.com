我用 HyperFrames 做了一个介绍 HyperFrames 的视频

发表于2026-06-15|更新于2026-06-15|技术
|浏览量:

用 HyperFrames 自举（dogfooding）——用框架本身来制作介绍它自己的视频。

动机

写完了 HyperFrames 的教程博客后，我想：为什么不直接做一个视频来展示它怎么用？正好验证一下这个「AI 友好、HTML 原生」的框架在实际操作中到底有多顺。

视频内容

30 秒，7 个场景：

时间 场景 内容
0s-4s 🎬 片头 HyperFrames 标题 + 特性标签
4s-8s ⚡ 快速开始 init / preview / render 三步
8s-13s 📝 合成结构 HTML + data 属性 + GSAP 示例
13s-17s 📐 三条铁律 Root / Timed / Animation
17s-21s 🤖 AI 优先 Skill 安装 + 自然语言描述
21s-26s ⌨️ CLI 命令 10 个常用命令一览
26s-30s 🚀 结束 链接引导

2. 编写合成 HTML

核心思路：每个场景是一个 div.slide.clip，用 data-start / data-duration / data-track-index 控制时间轴。

所有动画用 GSAP 注册：

三条铁律谨记：

  1. 根元素必须有 data-composition-id + data-width + data-height
  2. 每个要渲染的元素加 class="clip" + data-start / data-duration / data-track-index
  3. GSAP 动画必须 { paused: true }，注册到 window.__timelines

3. 校验

输出：0 errors, 2 warnings（GSAP Studio 编辑警告 + Google Fonts 建议，都不影响渲染）

4. 渲染

渲染统计：

指标 值
分辨率 1920×1080
时长 30s / 900 帧
码率 standard
渲染耗时 46.1s

渲染比视频本身还快（46s 渲染 30s 视频）——硬件加速 + 多线程并行抓帧。

视觉设计

暗色赛博风，跟博客主题一致：

  • 渐变标题：#00f0ff → #ff00aa
  • 背景光晕：模糊 120px 的彩色径向渐变
  • 代码块：半透明暗底 + 霓虹高亮
  • 卡片布局：四个规则 / AI 能力
  • 每个场景入场动画：淡入 + 位移 + 错位 stagger

遇到的问题

1. 国内网络

npm registry.npmjs.org 不通。切到 npmmirror 安装依赖，渲染时 Google Fonts 和 CDN 脚本自动缓存到 .cache/hyperframes/。

2. 路径问题

Windows 下 Hermes 的 write_file 路径解析有偏差，改用 Python 直接写入文件解决。

3. 时间轴对齐

GSAP 的时间偏移必须精确匹配 data-start。比如 Scene 3 在 8s 开始，tl.from(..., 8.2) 偏移 200ms 给上一场景淡出留缓冲。

总结

从 init 到产出 30s 视频，全部用 HTML + 文本编辑完成，没有打开任何视频剪辑软件。整个流程：

对于开发者来说，这种「写代码做视频」的体验远比拖拽时间线更自然。HyperFrames 的确定性和可重复性也让 CI/CD 集成成为可能——每次代码合并自动生成演示视频不再是幻想。
